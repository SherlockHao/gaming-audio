# AI 游戏音频生产与集成系统 MVP 需求清单 v0.1

## 1. 文档目标

本文件用于在 [AI_Game_Audio_PRD_v0.1.md](AI_Game_Audio_PRD_v0.1.md) 基础上，进一步定义第一阶段 MVP 的功能边界、优先级、交付内容、验收标准和里程碑，确保项目先打通最关键的端到端链路。

---

## 2. MVP 目标

MVP 目标不是追求全品类、全自动、高艺术质量，而是验证以下核心命题：

- 专业音频前置建制后，系统能否稳定接收低门槛需求输入
- 系统能否把标准化 SFX/UI/基础环境循环音从需求走到 Wwise 和 UE
- 系统能否通过自动化 QA 与 Profile 诊断发现典型问题
- 系统能否以可追踪、可回滚方式完成一轮真实项目迭代

---

## 3. MVP 成功定义

若满足以下条件，则认为 MVP 成功：

- 至少打通 1 条完整链路：需求输入 -> 素材生成 -> 检测处理 -> Wwise 导入 -> UE 挂接 -> QA 检测
- 支持至少 3 类标准化资产：`动作 SFX`、`UI 音效`、`简单环境循环`
- 支持至少 1 套项目模板：建议为 `动作游戏模板`
- 同一批次任务中，标准化需求自动处理成功率达到可内部试用水平
- 关键结果具备任务追踪、审批记录和回滚能力
- QA 能自动识别至少 5 类常见问题

---

## 4. MVP 范围

### 4.1 纳入范围

- 需求输入
- 视频/资源级语义理解
- Audio Intent Spec 生成
- 音频候选生成
- 基础后处理
- 基础 QC 检测
- Wwise 模板派生
- 音频导入 Wwise
- Event 自动创建
- SoundBank 生成
- UE AnimNotify 自动挂接
- 基础自动化 QA
- Profile 抓取与基础诊断
- 审批、追踪与回滚

### 4.2 不纳入范围

- 复杂音乐系统
- VO 流程
- 高级 Wwise 逻辑自动设计
- 复杂 Blueprint 逻辑自动接入
- 复杂 RTPC 自动设计
- 自动完成终混
- 自动完成项目后期精修

## 5. MVP 用户与角色分工

### 5.1 音频负责人

- 维护 Style Bible
- 定义 Category Rules
- 搭建 Wwise 模板
- 制定 UE mapping 规则
- 负责关键审批

### 5.2 音频技术设计师/TA

- 实现和维护自动化流程
- 维护导入规则、命名规则、模板版本
- 维护 QA 规则和诊断规则

### 5.3 策划

- 提交低门槛需求
- 提供视频、动画、美术特效或 UI 录屏
- 对结果做功能级确认

### 5.4 QA

- 运行自动化测试
- 标记异常时间点和对象
- 提交问题描述供系统诊断

## 6. MVP 核心用户故事

### US-01 提交动作音效需求

作为策划，我希望上传一个技能或动作视频，并只填写基础场景语义和播放方式，让系统自动产出对应声音并接入引擎。

### US-02 批量生产 UI 音效

作为策划，我希望批量提交一组 UI 操作音频需求，让系统自动按统一风格生成并导入。

### US-03 生成并接入基础环境循环音

作为内容制作人员，我希望提交一个机关或环境视频，让系统自动生成 loop 版本并挂接到游戏内容。

### US-04 批量导入 Wwise

作为音频技术设计师，我希望系统根据规则自动创建对象、导入音频、创建 Event 和生成 Bank。

### US-05 批量挂接 UE

作为音频技术设计师，我希望系统根据既有规则自动将 Event 挂到动画和基础组件上。

### US-06 自动检测常见接入问题

作为 QA，我希望自动化流程执行后，系统能告诉我哪些声音未触发、未发声、被 kill、被拉成 0 或资源未加载。

### US-07 问题自动诊断

作为 QA 或音频人员，我希望输入“某时间点某技能没声音”，系统能自动定位疑似根因。

## 7. MVP 功能清单

### 7.1 P0 功能

#### F-001 需求提交

- 支持创建任务
- 支持上传视频或引用动画资源
- 支持填写资源类型、场景语义、播放方式
- 支持添加基础标签和备注

**验收标准**

- 用户可在单页完成任务创建
- 任务创建后生成唯一 task_id
- 输入素材与元信息可被后续流程读取

#### F-002 需求结构化

- 将输入任务转为结构化需求对象
- 基于规则生成 Audio Intent Spec
- 输出置信度和待确认项

**验收标准**

- 每个任务都能生成结构化 spec
- 无法确定的字段会被标记，而不是静默失败

#### F-003 音频候选生成

- 根据 spec 生成至少 3 个候选音频
- 允许根据类别调用不同生成工作流

**验收标准**

- 任务可产出候选结果
- 候选结果有版本号和生成参数记录

#### F-004 基础后处理与 QC

- 自动切头尾
- 基础淡入淡出
- 分类响度统一
- 分类频段检测
- 输出 QC 报告

**验收标准**

- 输出导入版素材
- QC 结果包含通过/失败和原因

#### F-005 Wwise 模板派生

- 选择项目模板
- 根据任务类别派生目标路径和对象结构

**验收标准**

- 同类型任务进入固定规则路径
- 输出 Wwise object manifest

#### F-006 Wwise 导入与对象创建

- 导入音频
- 创建 Sound/Container/Event
- 设置基础命名与 Bus 指派

**验收标准**

- 导入成功后对象存在于预期路径
- Event 名称符合命名规则

#### F-007 SoundBank 生成

- 将目标对象加入指定 Bank
- 支持 Bank 生成
- 输出生成日志

**验收标准**

- 指定任务可生成可用 Bank
- 生成失败时日志可追踪

#### F-008 UE AnimNotify 自动挂接

- 根据规则为目标动画添加 Event 触发
- 输出 binding manifest

**验收标准**

- 可对指定动画批量挂接
- 无法匹配时进入 unresolved queue

#### F-009 任务追踪与审批

- 每个任务记录状态、版本、审批结果
- 支持音频审批和挂接审批

**验收标准**

- 能查看任务从创建到完成的完整状态
- 关键步骤有日志和责任信息

#### F-010 基础自动化 QA

- 接入自动化跑测
- 自动抓取 Wwise profile/capture
- 检测基础问题类型

**验收标准**

- 可基于自动脚本生成至少一份 QA 报告
- 报告包含问题时间点和关联对象

#### F-011 基础诊断

- 输入时间点和问题描述
- 输出疑似根因

**验收标准**

- 至少支持诊断以下问题：
- Event 未触发
- Voice 未创建
- Voice 被 Kill 或 Virtualize
- RTPC 拉低音量
- Bank/Media 缺失

### 7.2 P1 功能

#### F-012 UI 批量任务导入

- 支持表单或清单批量导入 UI 需求

#### F-013 环境循环音专用处理

- 支持 loop 检测与简单循环清洗

#### F-014 弱语义 UE 匹配

- 除规则匹配外，支持语义推断候选挂接

#### F-015 自动回滚

- 审批失败时自动恢复到前一版本

### 7.3 P2 功能

#### F-016 自动修复建议

- 对常见错误输出修复建议或候选操作

#### F-017 批量任务面板

- 支持批次管理、过滤、状态统计

## 10. MVP 验收标准

### 10.1 功能验收

- 可成功创建并执行标准化音频任务
- 可产出至少 3 类资产的可用版本
- 可将结果导入 Wwise 并生成 Event/Bank
- 可在 UE 中完成基础自动挂接
- 可执行自动化 QA 并产出问题报告

### 10.2 数据验收

- 每个任务具备唯一 ID
- 每个输出资产具备版本信息
- 每次 Wwise/UE 改动都有 manifest
- 每份 QA 报告可回溯到任务和对象

### 10.3 流程验收

- 任一关键步骤失败时，系统可标记失败原因
- 审批失败后可回退至上一可用状态
- 问题诊断链路能定位到至少一个疑似根因

---

## 11. MVP 依赖

- 已搭建基础 Wwise 工程模板
- 已接入 Unreal 与 Wwise 集成
- 已具备最小可用自动化跑测脚本
- 已准备至少一套动作游戏风格规则
- 已定义基础命名规则和路径规范

---

## 12. MVP 风险

- 视频理解能力不足会影响 spec 准确率
- 输入命名不规范会影响 UE 自动挂接
- QA 环节若拿不到稳定 profile，诊断效果会下降
- 生成资产一致性若不稳定，会影响团队信任度
- Wwise 模板若前期设计不扎实，后续难以扩展

---

- 可自动抓取 profile

- 可输出问题诊断报告

### M4：内部试运行

- 选定一个真实功能域试跑
- 收集自动化率、通过率、人工介入点
- 输出 MVP 复盘报告
